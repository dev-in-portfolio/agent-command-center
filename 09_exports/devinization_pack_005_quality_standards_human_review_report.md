## Purpose
This pack defines how the Agent Command Center reviews quality, applies standards, evaluates work, manages human approval, preserves evidence, governs deliverables, checks source freshness, and presents approval controls.

## What This Does Not Do
- Does not modify baseline family files
- Does not alter the 175-family lock
- Does not regenerate exports
- Does not hire or animate worker agents yet
- Does not build runtime yet
- Does not build UI yet

## Operating Doctrine
The Agent Command Center does not treat “created” as “quality checked.” A work product becomes ready only after it satisfies the objective, follows the correct standard, preserves evidence, passes applicable review gates, and is usable for the intended audience. Quality controls protect the whole command civilization without shrinking the art piece.
